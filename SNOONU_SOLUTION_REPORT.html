Executive Summary

The Problem

Snoonu's current greedy nearest-driver dispatch algorithm assigns each order to the closest available courier independently. While simple and fast, this approach ignores bundling opportunities where nearby orders could share routes, leading to higher fleet mileage, more drivers needed, and increased operational costs.

Our Solution

We developed an AI-Powered Adaptive Dispatch Engine that dynamically optimizes courier assignments using:

  • Route Bundling: Combine nearby orders (within 1.8km) into optimized multi-stop routes
  • Marginal Cost Bidding: Drivers bid based on incremental distance, not total route cost
  • TSP-PC Solver: Traveling Salesperson Problem with Precedence Constraints (pickup before dropoff)
  • Dynamic Strategy Switching: Automatically adapts between sequential and combinatorial modes based on order load

Technical Architecture

System Components

Core Engine (Python)
  • simulation.py - Discrete event simulator
  • dispatch.py - 4 dispatch strategies
  • scoring.py - Cost function with bundling incentives
  • models.py - Order, Driver, Bundle dataclasses
  • utils.py - Haversine distance, time utilities
Key Algorithms
  • TSP-PC Solver: Permutation enumeration with precedence
  • Marginal Cost Bidding: Incremental distance calculation
  • Combinatorial Bundle Generation: C(n,k) combinations
  • Adaptive Switching: Order rate threshold detection
